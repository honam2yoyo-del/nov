# 자동 Git 커밋 및 푸시 규칙

이 프로젝트에서 파일을 수정할 때마다 다음 단계를 **자동으로** 실행한다.

## 필수 규칙

코드나 파일을 수정한 후에는 사용자에게 별도로 확인을 구하지 않고 즉시 아래 순서대로 실행한다.

1. `git add -A` — 변경된 모든 파일을 스테이징
2. `git commit -m "<변경 내용을 요약한 메시지>"` — 커밋 메시지는 변경 사항을 간결하게 한 줄로 작성
3. `git push` — 원격 저장소(GitHub)에 푸시

## 커밋 메시지 형식

- 한국어 또는 영어 모두 허용
- 변경 내용을 명확히 설명하는 한 줄 메시지
- 예: `"로그인 버튼 스타일 수정"`, `"fix: typo in index.html"`

## 예외 사항

- `.env`, 시크릿 키, 비밀번호 등 민감한 정보가 포함된 파일은 커밋하지 않는다.
- 사용자가 명시적으로 "커밋하지 마" 또는 "푸시하지 마"라고 지시한 경우에는 따른다.
